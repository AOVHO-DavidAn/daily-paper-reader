## 摘要
大型语言模型（LLMs）通过思维链（CoT）提示在复杂推理任务中取得了显著成功。然而，这些模型常常表现出“计算过度思考”，生成冗余的推理步骤，增加延迟和成本而不提高准确性。最近的研究表明，CoT轨迹可以被大幅修剪，但现有方法通常依赖于强制静态思考预算、启发式过滤、通过分类进行次优早期退出或昂贵的重新训练。在本文中，我们介绍了OS-Pruner，一个轻量级插件框架，将思维链修剪形式化为一个最优停止问题。给定一个推理前缀，OS-Pruner通过优化一个权衡最终答案准确性与生成长度的显式效用，学习进一步推理是否值得其令牌成本。我们的新颖公式使模型能够动态评估推理链的足够终止点。OS-Pruner在训练和推理过程中都设计为轻量级，并为用户提供对推理努力与准确性权衡的细粒度控制。在多种推理基准测试和基础模型上，OS-Pruner实现了生成长度减少20-60%，而准确率损失极小。

## Abstract
Large Language Models (LLMs) have achieved remarkable success in complex reasoning tasks through Chain-of-Thought (CoT) prompting. However, these models often exhibit "computational overthinking," generating redundant reasoning steps that increase latency and cost without improving accuracy. Recent studies suggest that CoT trajectories can be significantly pruned, yet existing methods often rely on forcing a static thinking budget, heuristic filtering, sub-optimal early exit via classification, or expensive re-training. In this paper, we introduce OS-Pruner, a lightweight plug-in framework that formulates chain-of-thought pruning as an optimal stopping problem. Given a reasoning prefix, OS-Pruner learns whether further reasoning is worth its token cost by optimizing an explicit utility that trades off final-answer accuracy against generated length. Our novel formulation enables the model to dynamically assess the sufficient point of termination for a reasoning chain. OS-Pruner is designed to be lightweight during both training and inference, and to provide users with fine-grained control over the reasoning-effort vs. accuracy trade-off. On diverse reasoning benchmarks and base models, OS-Pruner achieves 20-60\% reduction in generation length with minimal accuracy sacrifice.

- **问题**：大型语言模型（LLM）通过链式思维（Chain-of-Thought, CoT）提示在复杂推理中表现出色，但常出现“计算过度思考”（computational overthinking）——生成冗余的推理步骤（如重复陈述、验证已确定的结论、低价值旁支计算等），导致延迟和成本增加，却未提升准确率，有时甚至降低准确率。
- **背景**：现有减轻过度思考的方法包括：1）预算强制/引导类方法（如 s1 的 budget forcing、Budget Guidance）——但预算多由外部设定而非依据推理状态动态调整；2）模型侧压缩方法（如长度惩罚微调、TokenSkip、步熵压缩等）——需昂贵重训练基座模型，且可能影响分布外任务；3）早期停止方法（如基于熵、置信度、收敛性的阈值分类、隐状态探针、验证器模型）——将停止问题简化为分类/阈值判定，未考虑继续推理的预期收益与成本的权衡。
- **整体含义**：作者论证 CoT 修剪本质上是一个序列决策问题——在每个推理步骤，系统需选择“停止并输出答案”或“继续推理并支付 token 成本”。因此，作者将其建模为**最优停止问题（Optimal Stopping Problem）**，并提出 OS-Pruner——一个轻量插件框架，动态决定推理链的终止点，在准确率与生成长度之间进行明确效用权衡。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 2.1 核心思想
- 将 CoT 修剪形式化为最优停止问题，学习一个参数化停止策略 π_θ(y_{≤i}|x) ∈ [0,1]，表示在给定推理前缀 y_{≤i} 后停止的概率。
- 奖励函数：`r(y_{≤i}|x) = A(y_{≤i}|x) - λ * L(y_{≤i})`，其中 `A` 为基于当前前缀回答的准确率（1正确/0错误），`L` 为前缀的 token 长度，`λ` 为权衡参数。
- 策略目标最大化期望奖励：`J(θ) = E_x[E_y[R_θ(x,y)]]`，其中 `R_θ` 为停时（stopping time）τ 下的期望奖励公式（3）。
- λ 的直观含义：为获得一个 token 的收益所需增加的准确率置信度。通过扫描 λ 值可绘制准确率-效率帕累托前沿。

### 2.2 关键技术细节

- **价值函数视角与分类子优性**：
  - 最优停止策略应满足贝尔曼方程（式5）：`Vλ = max{当前准确率, 继续推理的未折扣期望价值}`。停止条件是当前准确率 ≥ 继续价值 C_λ。
  - 分类阈值方法（式7）仅依赖固定阈值 γ 判断当前准确率是否足够，而忽略前缀的后续潜力。定理1证明：即使准确率已知，分类阈值方法与最优停止策略的期望奖励差距可任意大（K倍）。
- **数据预处理**：
  - 使用冻结的基座推理模型生成完整推理轨迹，对每一步前缀人为终止 CoT，并强制模型仅输出最终答案（通过提前追加 box 前缀），从而获得每个前缀的细粒度准确率标签。可一次性预计算，无需在策略更新时重新采样。
  - 利用 SGLang 的 fork 功能以树状请求并行计算。
- **策略架构**：
  - 停止策略头：一个线性层，输入为最后一层隐藏状态 h_l（最后一个推理步骤的隐藏状态），输出停止概率：`π_θ = σ(W^T h_l + b)`。
  - 微调基座模型的最后 n 个自注意力层（实验中 n=2），其余基座参数冻结。这使策略头轻量，推理时仅在段落边界调用，可并行预填充。
- **渐进训练**：
  - 对于较低的 λ，从零训练可能陷入“几乎总是继续至末尾”的局部最优。因此采用退火方案：从较高 λ 开始，周期性地降低 λ，如图1所示，训练中每轮退火后准确率和长度均有跳跃提升，最终在 λ 趋近0时准确率与长度差距缩小。

### 2.3 算法流程（算法1）
1. 输入问题 x，基座 LLM（最后 n 层微调），线性头 (W,b)
2. while 未停止 do
3.   生成下一个 token x_{t+1}
4.   if x_{t+1} 是段落结束符 then
5.     从缓存的第 L-n 层状态并行预填充最后 n 层，得到 h_l
6.     计算停止概率 π_θ = σ(W^T h_l + b)
7.     以概率 π_θ 停止
8.   end if
9. end while
10. 从 x_{1:t} 引出最终答案

## 6. 论文的主要结论与发现

- **帕累托最优**：OS-Pruner 在所有模型和基准上均位于或接近准确率-效率帕累托前沿，而固定阈值基线在长度预算减少时准确率下降更快。
- **长度缩减显著**：在低准确率牺牲（<2%）下，OS-Pruner 实现 20-60% 的生成长度缩减。最佳 AES 操作点上，DeepSeek-R1-Distill-Qwen-7B 在 GSM8K 上缩减 59.3%，MATH-500 上缩减 52.8%，准确率变化仅 -0.7 和 +2.7 百分点。
- **高难度任务更保守**：在 AIME 上，OS-Pruner 仅缩减 6.9% 长度，因为额外计算对难问题有价值。
- **模型侧压缩仍有改进空间**：即使已使用 DRPO 进行模型侧长度优化，OS-Pruner 仍能进一步减少长度并改善 AES（表1中 DRPO-7B 的最佳 AES 0.20 vs. FlashThink 0.09），说明最优停止方法可与模型侧方法互补。
- **理论优越性**：定理1证明分类阈值方法在理论上可任意劣于最优停止策略，实验验证了这一点。

## 7. 优点

- **轻量插件式设计**：无需重新训练或修改基座推理模型，仅添加一个小型策略头并微调最后 2 层，训练和推理都很高效。
- **动态决策**：停止决策基于当前推理状态（隐藏状态）及预期收益，而非预设预算或固定阈值，适应性更强。
- **细粒度控制**：通过单一超参数 λ 可调节准确率-效率权衡曲线，用户可根据需求选择操作点。
- **理论支撑**：提供了严格的最优停止公式，并证明了分类方法的次优性，为方法提供了理论基础。
- **实验设计公平**：与基线使用相同架构和微调规模，控制变量较好；考虑了指令遵循差异（强制答案消融）；评估包含多个难度和模型规模。

## 8. 不足与局限

- **应用范围有限**：仅测试了数学推理任务，未涉及编程、科学推理或常识问答等更广泛场景。CoT 过度思考在其他任务中是否类似需要验证。
- **模型规模局限**：最大实验模型为 20B 参数，未测试更大的前沿模型（如 70B、120B 或更大）。作者自身指出“将实验扩展到 20B 以上至关重要”。
- **架构和特征探索不足**：作者提到在架构和输入特征维度上的探索可能进一步提升性能，但本文未深入探索（如使用更多元特征如熵、收敛性等作为额外输入）。
- **强制答案的依赖**：对于指令遵循能力较弱的模型，必须在答案生成前强制 box 前缀，否则模型会继续在答案部分推理，削弱效果。这限制了方法在低指令遵循模型上的直接应用。
