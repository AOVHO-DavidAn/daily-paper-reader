## 摘要
跨图像比较推理对视觉语言模型（VLM）仍然具有挑战性，尤其是在正确预测需要细粒度属性定位和全局一致推理的情况下。我们提出了CoRe，一个针对该问题的统一框架。CoRe包括：(i) CoRe-20K，一个大规模三元组训练集，通过多专家协作流程从结构化视觉元数据自动构建，涵盖计数、深度、距离和空间关系；(ii) TriSR，一个结构化奖励框架，在GRPO优化下联合监督属性定位、判断对齐和三重一致性；(iii) CoRe-Bench，首个专用于细粒度跨图像比较推理的基准测试。实验表明，CoRe在CoRe-Bench上显著优于现有VLM，同时在标准多模态基准上保持竞争力，在部分准确率上比最强基线提高了28.2个百分点。

## Abstract
Cross-image comparative reasoning remains challenging for vision-language models (VLMs), especially when correct prediction requires fine-grained attribute grounding and globally consistent reasoning. We present CoRe, a unified framework for this problem. CoRe includes: (i) CoRe-20K, a large-scale triplet-based training set automatically constructed from structured visual metadata through a multi-expert collaborative pipeline, covering counting, depth, distance, and spatial relations; (ii) TriSR, a structured reward framework that jointly supervises attribute grounding, judgment alignment, and triplet consistency under GRPO optimization; and (iii) CoRe-Bench, the first benchmark dedicated to fine-grained cross-image comparative reasoning. Experiments show that CoRe substantially outperforms existing VLMs on CoRe-Bench while remaining competitive on standard multimodal benchmarks, achieving a 28.2-point gain in partial accuracy over the strongest baseline.

- **核心问题**：现有视觉语言模型（VLM）在处理**跨图像细粒度比较推理**时表现不佳。这类任务要求模型不仅理解单张图像的内容，还能在多张相关图像之间比较具体属性（如物体数量、场景深度、相对距离、空间布局），并保证推理链的全局一致性和可验证性。
- **背景与动机**：
  - 当前主流VLM（如GPT-4o、Gemini、Qwen3-VL）在常规多模态任务上表现优异，但在跨图像比较场景中频繁出现错误：如错误估计人数差值、颠倒深度顺序、误判距离变化、错认空间方位。
  - 现有基准（如MuirBench、MMRB）侧重语义级多图像理解（叙述、时序、检索），缺少对**细粒度度量比较**的覆盖。
  - 仅依赖最终答案正确性进行优化（outcome-only supervision）会导致模型通过捷径或不一致推理获得正确答案（论文分析显示，约17.1%的正确答案来自无效推理链）。
  - 使用LLM作为评判者（LLM-as-Judge）评估推理质量存在与人类判断的偏差，不适用于结构化正确性验证。
- **整体含义**：跨图像比较推理是VLM的一项独立且未被充分支持的能力，亟需专用训练数据、结构化监督信号和标准化评估基准。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程（文字说明）
### 核心思想
提出 **CoRe 框架**，包含三个核心组件：
- **CoRe-20K**：大规模、自动化构建的三元组（triplet）训练数据集。
- **TriSR**：结构化奖励框架，利用可验证的中间推理步骤进行联合监督。
- **CoRe-Bench**：首个专用细粒度跨图像比较推理基准。

### 关键技术细节
#### (1) 数据构建：多专家协作流水线
- **样本结构**：每个样本由三个相关图像（I₁, I₂, I₃）、任务维度（计数/深度/距离/空间关系）以及三对比较问题组成。
- **Metric Extraction Expert**：从结构化元数据（如实例计数、深度图、3D标注、边界框）中提取统一度量值 \(v_k\)。
- **Quality Control Expert**：通过噪声阈值过滤（\(\delta_{ij} \in [\theta_{\min}, \theta_{\max}]\)）排除不可靠或过于简单的三元组，并保证类别任务的多样性。
- **Question Generation Expert**：使用模板生成问题，随机打乱选项消除位置偏差。
- **生成规模**：20,000个三元组（每维度5,000个），对应60,000个比较问答对。保留4,000个（每维度1,000个）作为CoRe-Bench，16,000个用于训练。

#### (2) 结构化奖励框架 TriSR
- **结构化输出解析**：强制模型输出包含三部分：每图像属性估计 \(\{\hat{v}_k\}\)、推理链中的成对判断 \(\{\tilde{y}_{ij}\}\)、最终答案 \(\{\hat{y}_{ij}\}\)。
- **三种结构化奖励**：
  - **Attribute Grounding Reward** (\(R_{\text{ag}}\))：评估属性估计与元数据的偏差，使用指数衰减评分 \(s_m(\hat{v}_k, v_k) = \exp(-|\hat{v}_k - v_k| / \max(|v_k|, \epsilon))\)。
  - **Judgment Alignment Reward** (\(R_{\text{ja}}\))：同时衡量推理链判断与真实答案的匹配度、以及推理链判断与最终答案的一致性。
  - **Triplet Consistency Reward** (\(R_{\text{tc}}\))：验证三组成对比较结果是否满足逻辑传递性或区间兼容性（对数值任务使用区间叠加检查，对序数任务使用符号一致性检查）。
- **组合奖励**：\(R = R_{\text{task}} + \lambda_{\text{ag}} R_{\text{ag}} + \lambda_{\text{ja}} R_{\text{ja}} + \lambda_{\text{tc}} R_{\text{tc}}\)，其中 \(R_{\text{task}}\) 为最终答案准确率。
- **优化算法**：使用 **GRPO**（Group Relative Policy Optimization），对每组多个采样响应计算归一化优势，并加入KL正则化（\(\beta=0.01\)）防止策略漂移。

### 公式与算法流程（文字说明）
- 每个训练样本：\(x = (I_1, I_2, I_3, m, Q, A)\)，从元数据导出属性 \(a_i = f_m(I_i, A_i)\)，并由比较器 \(g_m\) 产生真实标签 \(y^*_{ij}\)。
- 模型输出 \(o = (t, \hat{y})\)，其中 \(t\) 为推理链，\(\hat{y}\) 为最终答案。
- 训练时，对每个查询采样 \(G\) 个响应，计算各自组合奖励，再计算组内归一化优势 \(\hat{A}_i\)，用clip后的重要性比率更新策略。
