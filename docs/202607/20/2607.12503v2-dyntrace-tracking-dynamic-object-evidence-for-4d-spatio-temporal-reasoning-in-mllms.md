## 摘要
4D时空推理，即联合建模3D空间结构和时间演化，对于理解动态世界并实现具身交互至关重要。尽管当前的多模态大语言模型（MLLMs）在静态场景理解和粗粒度4D任务中表现出强大的能力，但在连续动态场景感知方面仍存在显著局限性，特别是在追踪动态对象证据以实现连贯的4D时空推理方面。这一缺陷主要源于依赖稀疏的帧级观测，导致连续动态线索被碎片化，使模型无法区分真正的对象动态与相机诱导的表观运动。受人类在补偿视角变化的同时追踪动态线索的启发，我们提出了DynTrace，这是一个无需训练的4D时空推理框架，包含两个互补组件。动态轨迹可视化（DTV）将世界坐标轨迹重新投影到图像平面，提供基于几何信息的视觉先验，从而区分真正的对象动态与相机诱导的表观运动。同时，动态追踪标记（DT-Token）被组织成动态追踪图（DTG），追踪对象级动态线索、痕迹演化及关键时刻，为连贯的4D推理维护连续动态对象证据。这两个组件共同为MLLMs提供持续追踪的动态对象证据，这些证据基于几何信息视觉先验和结构化的时空痕迹。DynTrace持续改进开源MLLMs，在Dyn-Bench、VLM4D和DSI-Bench上取得了最先进的结果，验证了追踪动态对象证据对于鲁棒4D时空推理的重要性。

## Abstract
4D spatio-temporal reasoning, jointly modeling 3D spatial structure and temporal evolution, is essential for understanding dynamic worlds and enabling embodied interaction. While current Multimodal Large Language Models (MLLMs) show strong capabilities in static scene understanding and coarse-grained 4D tasks, they still have notable limitations in continuous dynamic scene perception, especially in tracking dynamic object evidence for coherent 4D spatio-temporal reasoning. This shortcoming stems mainly from relying on sparse frame-level observations, fragmenting continuous dynamic cues and leaving models unable to disentangle genuine object dynamics from camera-induced apparent motion. Inspired by humans tracking dynamic cues while compensating for viewpoint changes, we propose DynTrace, a training-free framework for 4D spatio-temporal reasoning with two complementary components. Dynamic Trajectory Visualization (DTV) reprojects world-coordinate trajectories onto the image plane, providing geometry-informed visual priors that disentangle genuine object dynamics from camera-induced apparent motion. Meanwhile, the Dynamic Trace Token (DT-Token), organized into a Dynamic Trace Graph (DTG), tracks object-level dynamic cues, trace evolution, and key moments, maintaining continuous dynamic object evidence for coherent 4D reasoning. Together, these two components equip MLLMs with continuously tracked dynamic object evidence, grounded in geometry-informed visual priors and structured spatio-temporal traces. DynTrace consistently improves open-source MLLMs, achieving state-of-the-art results on Dyn-Bench, VLM4D, and DSI-Bench, validating the importance of tracking dynamic object evidence for robust 4D spatio-temporal reasoning.

## 论文总结：DynTrace: Tracking Dynamic Object Evidence for 4D Spatio-Temporal Reasoning in MLLMs

### 1. 核心问题与整体含义（研究动机与背景）

- **核心问题**：当前的多模态大语言模型（MLLMs）在4D时空推理（联合建模3D空间结构与时间演化）中存在两大关键失败模式，导致其无法在连续动态场景中进行可靠推理：
    - **动态来源混淆 (Dynamic Source Confusion)**：模型无法区分物体的**真实运动**与**相机运动**引起的表观运动。当相机移动时，模型会错误地将相机运动归因于物体运动，从而扭曲对物体动态和关系的判断。
    - **动态痕迹丢失 (Dynamic Trace Loss)**：模型依赖稀疏的帧级采样，导致物体连续的运动轨迹被割裂成碎片。模型注意力会漂移到不相关的背景变化上，无法维持对动态物体的持续跟踪和证据链。
- **研究动机**：受人类4D认知机制的启发——人类能够同时进行自我运动补偿和实体追踪。Dyn-Bench等工作也验证了有效的动态场景推理依赖于对相机运动和物体动态的联合理解。
- **整体意义**：本文旨在解决MLLMs在连续动态场景理解中的**证据瓶颈**问题，即如何为模型提供明确、连续、可追踪的动态物体证据，而不仅仅是依赖稀疏的帧级观察。

### 2. 方法论

DynTrace是一个**无需训练**（Training-free）的推理框架，通过在推理阶段显式地为MLLMs提供动态物体证据来提升4D推理能力。该框架包含三个阶段和两个核心组件。

- **核心思想**：将原始视频转化为显式的动态证据，包括**几何感知的视觉先验**和**结构化的时空痕迹**，然后将其与用户查询一起输入MLLM进行推理。

- **技术细节与流程**：
    1.  **动态物体提取 (Dynamic Objects Extraction)**：
        - **查询引导的实体发现**：使用MLLM（如Qwen3-VL）根据用户查询分解出需要关注的视觉实体。
        - **相机运动消除**：利用WAFT光流估计器计算光流，通过拟合仿射变换提取**背景流**（由相机运动引起），然后从全光流中减去，得到**残差流**（代表独立物体运动）。
        - **动态物体聚类与跟踪**：基于残差流进行聚类，生成物体候选。结合文本提示和点提示，使用SAM3（Segment Anything Model 3）在视频中生成**时间一致的动态掩码**。

    2.  **时空动态编码 (Spatio-Temporal Dynamics Encoding)**：
        - **几何基础动态证据重建**：使用DA3（Depth Anything 3）估计每帧的深度和相机姿态。通过公式 `p_ti = T_t^{c→w} (z_ti * K_t^{-1} * [u_ti, v_ti, 1]^T)` 将每个动态物体的图像平面坐标投影到**共享的世界坐标系**，从而得到物体的真实3D轨迹。同时，还能计算出物体速度、方向、物体间距离、相对方位以及相机行为（平移、稳定性、缩放）。
        - **组件一：动态轨迹可视化 (Dynamic Trajectory Visualization, DTV)**：
            - 这是一个**视觉先验**分支。
            - 在世界坐标系中，从物体的当前轨迹提取一个短距离的运动方向，然后通过相机参数**重投影**回当前图像帧，在物体的掩码区域生成一个**红色方向箭头**。
            - **作用**：由于箭头源自世界坐标运动而非图像平面位移，它能清晰地将物体的真实运动方向与相机运动引起的表观运动区分开，直接缓解“动态来源混淆”问题。

        - **组件二：动态痕迹令牌与动态痕迹图 (Dynamic Trace Token (DT-Token) & Dynamic Trace Graph (DTG))**：
            - 这是一个**结构化的文本痕迹**分支。
            - 将视频划分为K个时间窗口。对于每个窗口中的物体，提取**动态线索**、**痕迹演化**和**关键时刻**，并将其转换为一个**物体侧的DT-Token**。
            - 对于物体对，同样提取**关系侧**的动态线索、痕迹演化和关键时刻，转换为**关系DT-Token**。
            - 将所有DT-Token（节点）和关系Token（边）与窗口的**相机总结**一起组织成一个窗口级图，并按时间序列化，形成最终的动态痕迹图（DTG）。
            - **作用**：DTG以紧凑的文本形式，显式地保留了长距离的动态物体轨迹、关系演变和关键事件，直接解决了“动态痕迹丢失”问题。

    3.  **表示整合与推理 (Representation Integration & Reasoning)**：
        - 将用户查询 `q`、DTV（`D`，带箭头的图像帧）和DTG（`G`，序列化的文本图）一起输入目标MLLM，模型直接基于这些丰富的动态证据进行最终回答。

### 8. 不足与局限

1.  **对细粒度部件级运动的局限**：论文指出，当前DynTrace将物体作为一个整体进行建模，对于需要关注身体部件（如手相对于狐狸的位置）等**更细致的部件级运动推理**场景，贡献有限（图9，案例(b)）。
2.  **对物体自身坐标系转换的局限**：当问题需要将运动转换为**物体自身的一人称视角**（如“相对于他/她/它的起始方向和位置，角色的位置是如何移动的？”）时，DynTrace的证据组织方式（围绕物体轨迹和相机-物体关系）可能不足以处理这种额外的坐标系变换（图9，案例(a)）。
3.  **计算开销增大**：尽管DynTrace无需训练，但其推理流程依赖于多个外部模型（MLLM用于实体发现、WAFT用于光流、SAM3用于分割、DA3用于深度/姿态），这相较于直接输入原始视频帧会引入**显著的计算和延迟开销**。论文未对此进行量化。
4.  **对中间模块的依赖**：尽管论文分析了其对中间预测误差的鲁棒性，但最终性能的上限仍受限于光流、分割、深度估计等中间任务模块的精度。极端情况下，如果这些模块产生严重错误，DynTrace的证据也会被污染。
